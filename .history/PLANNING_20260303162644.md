# My Wallet Backend & Feature Implementation Planning

## 📋 Proje Özeti
- Email + PIN tabanlı auth sistemi
- Offline-first mimari (local storage öncelikli)
- Smart senkronizasyon (veritabanı ile)
- OCR ile fiş okuma (n8n + OpenAI GPT-4o-mini)
- Döviz takibi (n8n + Exchange Rate API)
- **Tech Stack**: React Native, Node.js, Supabase (PostgreSQL), n8n

---

## 🎯 1. Database Design & Setup

### 1.1 Supabase Kurulumu
- [ ] Supabase projesi oluştur
- [ ] Veritabanı connection bilgilerini `.env` dosyasına ekle
- [ ] Supabase Client kütüphanelerini yükle (backend & mobile)

### 1.2 Database Schema
- [ ] **users** tablosu
  - `id` (UUID, primary key)
  - `email` (unique, not null)
  - `pin_hash` (encrypted, not null)
  - `created_at` (timestamp)
  - `last_login` (timestamp)
  
- [ ] **login_attempts** tablosu (🔴 CRITICAL: Brute Force Protection)
  - `id` (UUID, primary key)
  - `email` (string, index)
  - `attempt_count` (integer, default: 0)
  - `locked_until` (timestamp, nullable)
  - `last_attempt` (timestamp)
  
- [ ] **transactions** tablosu (mevcut local storage yapısı)
  - `id` (UUID, primary key)
  - `user_id` (foreign key -> users.id)
  - `type` (enum: 'income' | 'expense') - 🟡 HIGH: Database enum
  - `amount` (decimal, check: amount >= 0 AND amount <= 1000000000)
  - `currency` (default: 'TRY')
  - `category` (string, check: category IN (...)) - 🟡 HIGH: Enum validation
  - `description` (text, nullable)
  - `date` (date)
  - `created_at` (timestamp)
  - `updated_at` (timestamp)
  - `synced` (boolean, default: true)
  
- [ ] **exchange_rates** tablosu
  - `id` (serial, primary key)
  - `currency_code` (string, e.g., 'USD', 'EUR', 'XAU')
  - `rate_to_try` (decimal)
  - `last_updated` (timestamp)
  
- [ ] **receipts** tablosu (OCR için)
  - `id` (UUID, primary key)
  - `user_id` (foreign key -> users.id)
  - `transaction_id` (foreign key -> transactions.id, nullable)
  - `image_url` (text)
  - `file_size` (integer) - 🔴 CRITICAL: Size tracking
  - `mime_type` (string) - 🔴 CRITICAL: Validation
  - `ocr_data` (jsonb) - Ham OCR sonucu
  - `status` (enum: 'processing' | 'completed' | 'failed')
  - `created_at` (timestamp)

- [ ] **device_info** tablosu (senkronizasyon için)
  - `id` (UUID, primary key)
  - `user_id` (foreign key -> users.id)
  - `device_id` (unique string - cihaz identifier)
  - `last_sync_at` (timestamp)
  - `device_name` (string, optional)

### 1.3 Database Indexes (🔴 CRITICAL: Performance)
- [ ] **Performance-critical indexes:**
  ```sql
  CREATE INDEX idx_transactions_user_date ON transactions(user_id, date DESC);
  CREATE INDEX idx_transactions_user_category ON transactions(user_id, category);
  CREATE INDEX idx_receipts_user_status ON receipts(user_id, status);
  CREATE INDEX idx_device_info_user ON device_info(user_id, last_sync_at);
  CREATE INDEX idx_login_attempts_email ON login_attempts(email, locked_until);
  ```

### 1.4 Row Level Security (RLS) Policies
- [ ] Users sadece kendi verilerini görebilsin
- [ ] Her tablo için RLS policy oluştur

## 🔐 2. Authentication System

### 2.1 Backend Auth API (Node.js)
- [ ] `/api/auth/register` endpoint
  - Email validation (RFC 5322 compliant regex) - 🔴 CRITICAL
  - Email uniqueness kontrolü
  - PIN validation (6 digit numeric)
  - PIN encryption (bcrypt ile hash, cost: 12)
  - User kaydı oluştur
  - JWT token üret ve döndür (access + refresh) - 🟡 HIGH
  
- [ ] `/api/auth/login` endpoint (🔴 CRITICAL: Brute Force Protection)
  - Rate limiting: 5 req/min per IP
  - Login attempt kontrolü (5 failed = 15min lockout)
  - Email ile user bul
  - PIN doğrulama (bcrypt compare)
  - **Generic error message**: "Invalid credentials" (timing attack önleme) - 🟡 HIGH
  - JWT token üret (access: 30min, refresh: 7 days)
  - `last_login` güncelle
  - Device info kaydet/güncelle
  - Login attempts sıfırla
  
- [ ] `/api/auth/refresh` endpoint (🟡 HIGH: Token Refresh)
  - Refresh token doğrulama
  - Yeni access token üret
  - Refresh token rotation (optional)
  
- [ ] `/api/auth/verify-token` endpoint
  - JWT token doğrulama middleware
  - Expired token handling
  
- [ ] `/api/auth/change-pin` endpoint
  - Mevcut PIN doğrulama
  - Yeni PIN güncelleme
  - Email verification link gönder (security)

## 🔄 3. Sync Engine (Offline-First Architecture)

### 3.1 Smart Sync Strategy
- [ ] **Local Cache Kontrolü**
  - `last_sync_timestamp` local storage'a kaydet
  - Uygulama açılışta timestamp kontrol et
  - 5 dakikadan eskiyse sync yap, değilse lokal ver

- [ ] **Offline İşlem Yönetimi**
  - Local transaction'lara `synced: false` flag'i ekle
  - Pending sync queue oluştur
  - Background sync (ağ geldiğinde otomatik)

- [ ] **Conflict Resolution**
  - Server-side timestamping
  - Last-write-wins strategy (basit yaklaşım)
  - Gelecekte: conflict UI (kullanıcı seçsin)

### 3.2 Backend Sync API
- [ ] `POST /api/sync/push` - Lokal değişiklikleri server'a gönder (🔴 CRITICAL: Batch)
  - **Batch processing**: 50 transaction per request
  - Bulk insert/update desteği (Supabase .upsert())
  - Input validation (amount, category, date)
  - `synced: true` flag'ini güncelle
  - Rate limiting: 100 req/hour per user
  
- [ ] `GET /api/sync/pull` - Server'dan veri çek (🔴 CRITICAL: Pagination)
  - `since` parametresi (son sync zamanı)
  - **Pagination**: 100 items per page (cursor-based)
  - Sadece değişen kayıtları getir (delta sync)
  - User'ın tüm cihazlarından gelen verileri birleştir
  - Response compression (gzip)

- [ ] `GET /api/sync/full` - İlk kez sync (full data)
  - Yeni cihaz için tüm transaction geçmişi
  - **Pagination**: 100 items per page
  - Rate limiting: 1 req/hour

### 3.3 Frontend Sync Implementation
- [ ] **SQLite Integration** (React Native)
  - `expo-sqlite` veya `react-native-sqlite-storage`
  - Local database schema (transactions tablosu)
  - CRUD operations
  
- [ ] **AsyncStorage → SQLite Migration** (🔴 CRITICAL: Data Safety)
  ```javascript
  // Migration script
  async function migrateToSQLite() {
    const version = await AsyncStorage.getItem('migration_version');
    if (version === '2.0') return;
    
    const transactions = await AsyncStorage.getItem('transactions');
    if (transactions) {
      const parsed = JSON.parse(transactions);
      // Date deserialization (ISO string → Date)
      await SQLite.insertBatch(parsed);
      await AsyncStorage.removeItem('transactions');
    }
    
    await AsyncStorage.setItem('migration_version', '2.0');
  }
  ```
  
- [ ] **Sync Service** (🟡 HIGH: Retry Logic)
  - Network state listener (online/offline detection)
  - Auto-sync on app foreground
  - Manual sync button (settings'te)
  - **Exponential backoff retry**: 1s, 2s, 4s, 8s, 16s
  - **Batch sync**: 50 transactions per request
  - Sync status indicator (UI)
  - Priority queue (recent transactions first)

- [ ] **Yeni Cihaz İlk Giriş**
  1. PIN doğrulama
  2. Local storage boş mu kontrol et
  3. `GET /api/sync/full` ile tüm veriyi çek (paginated)
  4. Local SQLite'a yaz
  5. `last_sync_timestamp` kaydet
  6. Success notification

## 📸 4. OCR Feature (Fiş Okuma)

### 4.1 n8n Workflow Setup
- [ ] n8n kurulumu (self-hosted veya cloud)
- [ ] **Workflow Node'ları:**
  1. [ ] Webhook node (trigger)
     - `POST /webhook/ocr-receipt`
     - Body: `{ "image": "base64_string", "user_id": "uuid" }`
  
  2. [ ] OpenAI node (GPT-4o-mini)
     - Vision API kullanımı
     - Prompt: 
       ```
       Bu fiş görselini analiz et. Şu bilgileri JSON formatında çıkar:
       {
         "total_amount": number,
         "currency": "TRY",
         "date": "YYYY-MM-DD",
         "category": "Market|Restaurant|Fuel|Other",
         "merchant_name": string
       }
       ```
  
  3. [ ] Code node (data transformation)
     - OpenAI response'u parse et
     - Veritabanı formatına dönüştür
  
  4. [ ] HTTP Request node (Backend API)
     - `POST /api/transactions/create-from-ocr`
     - OCR verisini transaction olarak kaydet
  
  5. [ ] Webhook Response
     - Success/error mesajı döndür

### 4.2 Backend OCR API
- [ ] `POST /api/receipts/upload` (🔴 CRITICAL: File Upload Security)
  - Rate limiting: 10 req/hour per user
  - **File size limit**: Max 10MB per image
  - **MIME type validation**: Whitelist (image/jpeg, image/png, image/heic)
  - **Magic number check**: Verify actual file type (not just extension)
  - **Image optimization** (🟡 HIGH):
    - Compress with sharp (80% quality JPEG)
    - Generate thumbnail (200x200px)
    - Strip EXIF metadata (privacy)
    - Max dimension: 4096x4096px
  - S3 veya Supabase Storage'a kaydet
  - n8n webhook'una base64 gönderi
  - Receipt record oluştur (status: processing)
  
- [ ] `POST /api/transactions/create-from-ocr` (n8n'den çağrılacak)
  - OCR verisini transaction'a çevir
  - Transaction kaydet
  - Receipt status'ü completed yap
  - User'a push notification (opsiyonel)
  
- [ ] `GET /api/receipts/:id` - Receipt detayı
- [ ] `GET /api/receipts` - User'ın tüm receipts'leri

### 4.3 Frontend OCR UI
- [ ] **Fiş Ekle Butonu** (Home/Create screen)
- [ ] **Kamera/Galeri Seçimi**
  - `expo-image-picker` veya `react-native-image-picker`
  - Image preview
  
- [ ] **OCR Processing UI**
  - Loading indicator

- [ ] **AsyncStorage → SQLite Migration**: Data loss prevention
- [ ] **Pagination Strategy**: Memory optimization (100 items/page)
- [ ] **Rate Limiting**: DDoS protection (per endpoint limits)
- [ ] **Input Validation**: SQL injection, XSS prevention
- [ ] **Batch Sync**: Network efficiency (50 items/batch)

### 🟡 HIGH (Should Have - Phase 2)
- [ ] **Token Refresh Mechanism**: Session management

## ⚠️ Önemli Notlar

### Senkronizasyon Stratejisi Detayı:
1. **İlk açılış (yeni cihaz)**:
   - Local storage boş
   - Email + PIN ile login
   - Backend: `GET /api/sync/full`
   - Tüm transaction'lar local SQLite'a yazılır
   - `last_sync_timestamp` = şimdi

2. **Sonraki açılışlar**:
   - `last_sync_timestamp` kontrol
   - 5 dk'dan yeniyse: Local verilerle devam
   - 5 dk'dan eskiyse: Delta sync (`GET /api/sync/pull?since=timestamp`)
   - Background'da pending transactions'ları push et

3. **Offline işlemler**:
   - Tüm CRUD işlemleri local SQLite'da
   - `synced: false` flag'i
   - Online olunca otomatik `POST /api/sync/push`

4. **Conflict çözümü** (şimdilik basit):
   - Server timestamp > local timestamp ise server kazanır
   - Gelecekte: Kullanıcıya sor

### Güvenlik Önlemleri:
- PIN'ler bcrypt ile hash'lenecek (ASLA plain text saklanmayacak, cost: 12)
- JWT token expiry: access 30dk, refresh 7 gün
- **Rate limiting (express-rate-limit):**
  - Login: 5 req/min per IP
  - Register: 3 req/hour per IP
  - Upload: 10 req/hour per user
  - Sync: 100 req/hour per user
- **Input validation:**
  - Email: RFC 5322 compliant
  - PIN: 6 digits (000000-999999)
  - Amount: 0 ≤ x ≤ 1,000,000,000
  - Category: Enum match with constants
  - Description: Max 500 chars, sanitized
- **File upload security:**
  - Max 10MB per image
  - MIME whitelist: image/jpeg, image/png, image/heic
  - Magic number verification
  - EXIF metadata stripping
- **Error handling:**
  - Generic error messages (no info leakage)
  - "Invalid credentials" for all auth failures
  - Structured logging with field redaction
- HTTPS zorunlu (production)
- SQL injection önleme (Supabase parameterized queries)
- Secure storage: expo-secure-store (AES-256) for JWT tokens

### n8n Best Practices:
- Error handling: Her node'da error output tanımla
- Retry logic: HTTP request'lerde 3 retry
- Logging: Code node'da console.log yerine workflow log kullan
- Webhook güvenliği: API key veya signature verification
